# 新闻自动分析与发送 Skill

## 工作流程

### 步骤1：读取状态文件

读取 `memory/news-analysis-state.json` 获取上次分析的状态：
- `lastAnalyzedFile`: 上次最后分析的文件路径
- `totalAnalyzed`: 已分析的新闻总数
- `lastAnalysisTime`: 上次分析时间

如果状态文件不存在，则初始化状态，表示从头开始分析。

### 步骤2：扫描新闻目录

扫描 `input/data/articles/` 目录下的所有新闻文件。

**目录结构：**
```
input/data/articles/
├── 2026-03-07/
│   └── 20260307-083703_0005/
│       └── 01-xxxxxxxxx.md
├── 2026-03-08/
│   ├── 20260308-083703_0005/
│   │   └── 01-xxxxxxxxx.md
│   └── 20260308-190008_0006/
│       └── 01-xxxxxxxxx.md
```

**文件排序规则：**
按文件夹名称排序（日期+时间格式），新文件在后。
排序逻辑：
1. 按日期目录排序（YYYY-MM-DD）
2. 同一日期内按批次目录排序（YYYYMMDDHHMMSS_XXXX）
3. 同一批次内按文件名排序（01-, 02- 等）

### 步骤3：识别新增新闻

比较当前扫描到的文件列表与状态文件中记录的 `lastAnalyzedFile`：
- 找出所有在 `lastAnalyzedFile` 之后的新文件
- 按时间顺序排列（旧的在前，新的在后）

**如果没有新增新闻：**
直接告知用户"暂无新增新闻需要分析"，任务结束。

**例外情况：**
如果用户明确说"分析所有新闻"或"重新分析"，则无视状态文件，分析所有新闻。

### 步骤4：分析新增新闻

对每一条新增新闻，依次执行：

#### 4.1 读取新闻内容

使用 Read 工具读取新闻 Markdown 文件，提取：
- 标题（H1标题）
- 元数据（source, show_time, url 等）
- 正文内容

#### 4.2 调用股票分析

使用 `stock-analysis` skill 的分析能力（即按照 stock-analysis/SKILL.md 的指示），以该新闻为输入生成三视角股票分析报告。

**重要原则：**
- 新闻之间默认是无关的，**不要汇总**多份报告
- 逐条分析，完成一条、发送一条，再分析下一条
- 每条新闻生成独立的分析报告
- 报告需保存到 `memory/stock-analysis/YYYY-MM-DD/` 目录

### 步骤5：分条发送分析报告

每条新闻分析完成后，**将报告拆分成多条消息发送**，而不是一次性发送完整报告。

**发送流程（对每条新闻）：**
1. 分析新闻内容
2. 生成完整的三视角报告
3. **按以下顺序分5条消息发送：**

   - **第1条：📰 新闻摘要**
   - **第2条：一、价值守护者视角（长期价值投资）**
   - **第3条：二、波段猎手视角（技术分析趋势）**
   - **第4条：三、热点感知器视角（短线情绪博弈）**
   - **第5条：四、综合研判与策略建议**

4. 等待所有消息发送完成
5. 继续处理下一条新闻

**默认发送目标：**
- 如果没有特别指定，询问用户发送给谁
- 可以发送到联系人或群聊
- 询问一次后，后续新闻默认发送到同一目标

**发送方式：**
运行脚本（每条消息单独发送）：
```bash
uv run python .claude/skills/wechat-send-fixed-message/scripts/send_fixed_message.py \
  --target "{target_name}" \
  --message "消息内容"
```

### 步骤6：更新状态文件

分析完成后，更新 `memory/news-analysis-state.json`：
- `lastAnalyzedFile`: 最后分析的文件路径
- `totalAnalyzed`: 累计分析总数
- `lastAnalysisTime`: 本次分析时间（ISO 8601格式，北京时间）

## 执行流程示例

**用户输入：** "检查有没有新新闻要分析"

**执行步骤：**
1. 读取 `memory/news-analysis-state.json`
2. 扫描 `input/data/articles/` 目录
3. 发现 2 条新增新闻
4. 询问用户："发现 2 条新增新闻，发送到哪里？"
5. 用户回复："发到'投资交流群'"
6. **分析并分条发送第1条新闻：**
   - 分析新闻内容
   - 生成三视角报告
   - 分5条消息发送到"投资交流群"：
     1. 📰 新闻摘要
     2. 一、价值守护者视角
     3. 二、波段猎手视角
     4. 三、热点感知器视角
     5. 四、综合研判与策略建议
   - 等待发送完成
7. **分析并分条发送第2条新闻：**
   - 分析新闻内容
   - 生成三视角报告
   - 分5条消息发送到"投资交流群"
   - 等待发送完成
8. 更新状态文件
9. 告知用户："已完成 2 条新闻的分析和发送"

## 状态文件格式

`memory/news-analysis-state.json`:
```json
{
  "lastAnalyzedFile": "/Users/.../input/data/articles/2026-03-08/20260308-190008_0006/01-202603083665356921.md",
  "totalAnalyzed": 6,
  "lastAnalysisTime": "2026-03-08T21:15:00+08:00"
}
```

## 注意事项

1. **增量分析**：只分析新增的新闻，避免重复分析
2. **独立处理**：新闻之间默认是无关的，**不要汇总**报告，每条新闻单独分析和发送
3. **逐条发送**：分析完一条就立即发送，不要等所有新闻分析完成再一起发送
4. **分条消息**：每条新闻的分析报告必须拆分成5条独立消息发送（摘要+3个视角+综合）
5. **状态持久化**：每次分析后必须更新状态文件
6. **错误处理**：如果某条新闻分析失败，记录错误继续分析下一条
7. **发送确认**：发送完成后确认用户是否收到
8. **时间格式**：所有时间使用北京时间（UTC+8），ISO 8601格式

## 特殊情况处理

**首次运行（无状态文件）：**
- 提示用户这是首次运行
- 询问是否分析所有现存新闻
- 或只分析最近N条

**新闻格式异常：**
- 跳过无法解析的文件
- 记录到错误日志

**发送失败：**
- 保存报告到本地文件
- 告知用户手动发送
- 仍更新状态（分析已完成）

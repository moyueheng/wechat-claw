# 新闻自动分析与发送 Skill

## 工作流程

### 步骤1：扫描新闻目录

扫描 `input/data/articles/` 目录下的所有新闻文件。

**目录结构：**
```
input/data/articles/
├── 2026-03-07/
│   └── 20260307-083703_0005/
│       └── 01-xxxxxxxxx.md
├── 2026-03-08/
│   ├── 20260308-083703_0005/
│   │   └── 01-xxxxxxxxx.md
│   └── 20260308-190008_0006/
│       └── 01-xxxxxxxxx.md
```

**文件排序规则：**
按文件夹名称排序（日期+时间格式），新文件在后。
排序逻辑：
1. 按日期目录排序（YYYY-MM-DD）
2. 同一日期内按批次目录排序（YYYYMMDDHHMMSS_XXXX）
3. 同一批次内按文件名排序（01-, 02- 等）

### 步骤2：识别待分析新闻

扫描 `input/data/articles/` 目录下的所有 `.md` 文件。

**重要**： 按时间顺序处理新闻

- 找出所有未归档的新闻文件
- 按时间顺序排列（旧的在前，新的在后）

**如果没有新闻需要分析：**
直接告知用户"暂无新增新闻需要分析"，任务结束。

**例外情况：**
如果用户明确说"分析所有新闻"或"重新分析"，则分析所有现存新闻。

### 步骤3：分析新闻

对每一条待分析新闻，依次执行：

#### 3.1 读取新闻内容

使用 Read 工具读取新闻 Markdown 文件，提取：
- 标题（H1标题）
- 元数据（source, show_time, url 等）
- 正文内容

#### 3.2 调用股票分析

使用 `stock-analysis` skill 的分析能力（即按照 stock-analysis/SKILL.md 的指示），以该新闻为输入生成三视角股票分析报告。

**重要原则：**
- 新闻之间默认是无关的，**不要汇总**多份报告
- 逐条分析，完成一条、发送一条，再分析下一条
- 每条新闻生成独立的分析报告
- 报告需保存到 `memory/stock-analysis/YYYY-MM-DD/` 目录

### 步骤4：分条发送分析报告到飞书

每条新闻分析完成后，**将报告拆分成多条飞书消息发送**，而不是一次性发送完整报告。

**发送流程（对每条新闻）：**
1. 分析新闻内容
2. 生成完整的三视角报告
3. **按以下顺序分5条消息发送， 五条消息要和 stock-analysis 分析的内容一字不落的分段落发送， 保证完整：**

   - **第1条：📰 新闻摘要** 
   - **第2条：一、价值守护者视角（长期价值投资）**
   - **第3条：二、波段猎手视角（技术分析趋势）**
   - **第4条：三、热点感知器视角（短线情绪博弈）**
   - **第5条：四、综合研判与策略建议**

4. 等待所有消息发送完成
5. **归档已分析的新闻文件**

   将已分析的新闻文件移动到归档目录：
   ```bash
   mkdir -p input/data/archived/$(date +%Y-%m-%d)
   mv "{news_file_path}" "input/data/archived/$(date +%Y-%m-%d)/"
   ```

6. 继续处理下一条新闻

**默认发送目标：**
- 如果用户明确指定飞书用户名或群名，按用户指定发送
- 如果未明确指定，则优先使用仓库根目录 `.env` 中已配置的飞书别名映射
- 推荐使用：
  - `FEISHU_USER_ALIAS_JSON`
  - `FEISHU_CHAT_ALIAS_JSON`
- 如果既没有用户明确指定，也没有可用的飞书别名映射，直接报错并停止任务
- 不再询问是否发送到微信，也不允许回退到微信发送

**发送方式：**
运行脚本（每条消息单独发送）：
```bash
uv run --with lark-oapi python .agents/skills/feishu-send-message/scripts/send_message.py \
  --chat-name "{target_name}" \
  --message "消息内容"
```

发送给单个飞书用户时：
```bash
uv run --with lark-oapi python .agents/skills/feishu-send-message/scripts/send_message.py \
  --user-name "{target_name}" \
  --message "消息内容"
```

## 执行流程示例

**用户输入：** "检查有没有新新闻要分析"

**执行步骤：**
1. 扫描 `input/data/articles/` 目录（排除 archived 子目录）
2. 发现 2 条待分析新闻
3. 确认飞书发送目标："投资交流群"
4. 若用户未指定，则尝试从 `.env` 的飞书别名映射中解析目标
5. **分析并分条发送第1条新闻：**
   - 分析新闻内容
   - 生成三视角报告
   - 分5条飞书消息发送到"投资交流群"：
     1. 📰 新闻摘要
     2. 一、价值守护者视角
     3. 二、波段猎手视角
     4. 三、热点感知器视角
     5. 四、综合研判与策略建议
   - 等待发送完成
   - 将该新闻文件移动到 `input/data/archived/2026-03-08/`
6. **分析并分条发送第2条新闻：**
   - 分析新闻内容
   - 生成三视角报告
   - 分5条飞书消息发送到"投资交流群"
   - 等待发送完成
   - 将该新闻文件移动到 `input/data/archived/2026-03-08/`
7. 告知用户："已完成 2 条新闻的分析和发送" **保证所有的新闻的发送完毕，并进行归档**

已分析的新闻将被移动到归档目录：
```
input/data/archived/
├── 2026-03-07/
│   └── 20260307-083703_0005/
│       └── 01-xxxxxxxxx.md
├── 2026-03-08/
│   ├── 20260308-083703_0005/
│   │   └── 01-xxxxxxxxx.md
│   └── 20260308-190008_0006/
│       └── 01-xxxxxxxxx.md
```

## 注意事项

1. **文件归档**：分析完成的新闻会被移动到归档目录，避免重复分析
2. **独立处理**：新闻之间默认是无关的，**不要汇总**报告，每条新闻单独分析和发送
3. **逐条发送**：分析完一条就立即发送，不要等所有新闻分析完成再一起发送
4. **分条消息**：每条新闻的分析报告必须拆分成5条独立消息发送（摘要+3个视角+综合）
5. **错误处理**：如果某条新闻分析失败，记录错误继续分析下一条
6. **发送确认**：发送完成后确认用户是否收到
7. **时间格式**：所有时间使用北京时间（UTC+8），ISO 8601格式
8. **归档管理**：定期检查归档目录，可按日期清理旧文件， 每处理一个新闻必须归档后再处理下一个
9. **发送通道**：最终报告只允许发送到飞书，不允许发送到微信

## 特殊情况处理

**首次运行：**
- 扫描所有现存新闻文件
- 询问是否分析所有新闻或只分析最近N条

**新闻格式异常：**
- 跳过无法解析的文件
- 记录到错误日志
- 不移动该文件到归档目录

**发送失败：**
- 保存报告到本地文件
- 告知用户手动发送
- 仍归档新闻文件（分析已完成）

**飞书目标未配置：**
- 如果没有显式目标，且 `.env` 中没有可用的飞书别名映射，直接报错并停止
- 不允许临时改走微信

**归档失败：**
- 如果无法创建归档目录或移动文件，记录错误但不中断流程
- 下次运行时可能会重新分析该新闻
